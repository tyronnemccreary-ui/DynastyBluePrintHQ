# Coach Experience

**Product:** Dynasty Blueprint HQ  
**Purpose:** Coach Journey and Experience Blueprint  
**Audience:** Product direction, future feature planning, and experience design  
**Version:** 1.0

---

# Mission

Dynasty Blueprint HQ should make every user feel like they are running a real football program.

The experience should feel less like opening a tool and more like walking into a Football Operations Center where the staff has already prepared the briefing, organized the data, identified the risks, and highlighted the next decisions.

The coach should never feel alone with a spreadsheet.

The coach should feel supported by a focused, intelligent, disciplined operations department.

The experience we are trying to create is:

> Confident leadership through clear football decisions.

Dynasty Blueprint HQ should help the coach understand the program at every stage:

- What is happening?
- Why does it matter?
- What decision should I consider next?

The application should not make decisions for the coach.

It should prepare the coach to make better decisions.

---

# The Coach's Journey

The coach journey is the emotional and operational backbone of Dynasty Blueprint HQ.

The application should follow the rhythm of a dynasty, not the structure of a database.

The coach moves through a football life cycle:

Accept New Job  
Welcome to the Program  
Football Operations Setup  
Program Assessment  
Executive Briefing  
Preseason  
Recruiting Preparation  
Week One  
Weekly Operations  
Midseason  
Championship Push  
Players Leaving  
Transfer Portal  
Offseason Planning  
Year Two  
Multiple Seasons

Each stage has a different emotional state, different available decisions, and different intelligence needs.

The product should respond to that rhythm.

# Accept New Job

## What the Coach Is Trying to Accomplish

The coach is stepping into a new role.

They are trying to understand:

- What kind of program did I inherit?
- What are the expectations?
- What resources do I have?
- What kind of rebuild, reload, or championship push is this?

This is the first emotional moment of the dynasty.

The coach should feel like they are being welcomed into a real program.

## What the Application Should Display

The application should display a focused introduction to the program setup process.

It should not overwhelm the coach with every future department.

It should show:

- Start New Dynasty
- Continue Existing Dynasty
- School selection
- Coach identity
- Coach role
- Basic program creation

## What Recommendations Should Appear

At this stage, recommendations should be minimal.

The application should not pretend to know the program before the coach has selected a school.

Useful guidance may include:

- Select the team you are taking over.
- Create your coach profile.
- Begin Program Assessment.

## What Departments Become Active

Active:

- Welcome to the Program

Pending:

- Program Office
- War Room
- Football Operations Budget
- Roster Intelligence
- Recruiting Command Center
- Transfer Portal
- Staff Management

## What Intelligence Should Be Generated

Only basic setup intelligence should exist.

The application should know:

- Dynasty type
- Coach name
- Coach role
- Selected school

## What Decisions Should Be Made

The coach decides:

- New dynasty or existing dynasty
- School
- Coach name
- Coach role

# Welcome to the Program

## What the Coach Is Trying to Accomplish

The coach is officially starting the program.

They want the app to recognize the team, establish context, and prepare the football operations environment.

The coach should not feel like they are filling out administrative paperwork.

They should feel like they are taking over an athletic department.

## What the Application Should Display

The application should display:

- Welcome message
- Dynasty type selection
- Searchable school selector
- School name
- Conference
- Coach name
- Coach role
- Start button

The experience should be clean, direct, and premium.

The school selector should be fast and respectful of the coach's time.

## What Recommendations Should Appear

Recommendations should remain light.

The product should guide setup, not analyze yet.

Possible setup guidance:

- Select your school to activate Program Office.
- Create your coach profile to unlock the War Room.
- Continue to Program Assessment after setup.

## What Departments Become Active

After the coach completes setup:

Ready:

- Program Office
- War Room
- Football Operations Budget if Dynasty Points are available
- Screenshot Import

Needs Activation:

- Roster Intelligence
- Recruiting Command Center
- Staff Management

Season Locked:

- Transfer Portal unless offseason

## What Intelligence Should Be Generated

The application should create a Program Operations Profile.

It should clone school facts from the Football Knowledge Base into the user's living program profile.

The Master Football Knowledge Base remains unchanged.

Generated intelligence:

- Program identity
- Initial department activation state
- Initial budget state
- Initial War Room availability

## What Decisions Should Be Made

The coach decides:

- Which school to manage
- What role they are playing
- Whether to enter Program Office or War Room first

# Football Operations Setup

## What the Coach Is Trying to Accomplish

The coach is preparing the program for real decision support.

They want to add the information that makes recommendations stronger.

The coach should understand that the application can already provide basic context from the selected school, but stronger insights require dynasty-specific information.

## What the Application Should Display

The application should show department activation clearly.

Each department should explain:

- Whether it is ready
- What data it needs
- Why that data matters
- What the coach can do next

The product should avoid making missing data feel like an error.

Missing data is a setup state.

## What Recommendations Should Appear

Recommendations should focus on setup priorities:

- Upload roster data to activate Roster Intelligence.
- Add recruiting board data before evaluating class strategy.
- Add staff data before making hiring recommendations.
- Review Football Operations Budget before Week One.

## What Departments Become Active

Ready:

- Program Office
- War Room
- Football Operations Budget
- Screenshot Import

Needs Activation:

- Roster Intelligence
- Recruiting Command Center
- Staff Management

Conditional:

- Transfer Portal based on dynasty phase

## What Intelligence Should Be Generated

The application should generate:

- Department activation map
- Basic Program Health
- Initial budget posture
- Setup recommendations

## What Decisions Should Be Made

The coach decides:

- Which data to add first
- Whether to begin with roster, recruiting, staff, or budget
- Whether the program is ready for weekly operations

# Executive Briefing

## What the Coach Is Trying to Accomplish

The coach wants the shortest path to clarity.

They want to know:

- How is my program doing?
- What matters right now?
- What should I do next?

The War Room should become the coach's home base.

## What the Application Should Display

The War Room should display:

- Program Status
- Program Health
- Executive Summary
- Department Reports
- Budget Summary
- Roster Alert when available
- Top Three Decisions
- Department Activation
- Quick Actions

The War Room should never become cluttered.

It should feel like an executive briefing, not a dashboard dump.

## What Recommendations Should Appear

Only the top three decisions should appear.

They should come from the Football Operations Engine.

Recommendations should include:

- Recommendation
- Reason
- Tradeoffs
- Risk if ignored
- Expected Outcome
- ROI
- Priority
- Time Horizon

## What Departments Become Active

The War Room is active once a Program Operations Profile exists.

It should show the status of all departments.

## What Intelligence Should Be Generated

Generated intelligence:

- Executive Summary
- Department reports
- Top three recommendations
- Budget health
- Program health
- Activation status

## What Decisions Should Be Made

The coach decides:

- Which top recommendation to act on
- Which department to open
- Whether to add missing data
- Whether to adjust budget or recruiting priorities

The coach is moving from planning into execution.

Week One is the point where some strategic decisions should become locked or harder to change.

The coach should understand what is still flexible and what is now committed.

## What the Application Should Display

The application should display:

- Budget lock status
- Weekly Executive Briefing
- Top three decisions

- Roster alert
- Department activation

# Program Operations Setup

Program Operations Setup should explain the difference between preloaded school data and user dynasty data.

The coach should understand:

- The app knows school facts from the Master Team Database.
- The app needs roster, recruiting, staff, and portal data to provide stronger live recommendations.
- Missing data is normal at the beginning.

Setup should feel like preparing the operations department.

It should not feel like fixing errors.

The application should use department activation states:

- Ready
- Needs Activation
- Season Locked

Each activation state should explain what it means and what action is available.

The War Room is the home screen.

The War Room should answer:

- How is my program doing?
- What should I do next?
- What decisions require attention?

It should not feel like a statistics page.

The War Room should prioritize:

- Program clarity
- Executive summary

# Product Experience Standards

## Every Upload Should Immediately Produce Value

When the coach uploads a roster, recruiting board, staff screen, or screenshot, the application should respond with clear value.

Even if full analysis is not available, the app should confirm:

- What was uploaded
- What department it supports
- What can be shown now
- What analysis is coming later

The coach should never see an upload stuck forever.

## Every Recommendation Should Be Actionable

A recommendation should tell the coach what to consider doing.

It should not be vague.

It should include:

- Recommendation
- Reason
- Tradeoff
- Risk if ignored
- Expected Outcome
- ROI
- Priority
- Time Horizon

## Every Page Should Answer Three Questions

Every page should answer:

1. What happened?
2. Why?
3. What should I do next?

If a page cannot answer those questions, it should be refined.

## The Application Should Never Make the Coach Wonder What To Do

The coach may choose not to follow a recommendation.

That is healthy.

But the coach should never wonder what the application is trying to tell them.

Every page should have a clear purpose.

Every recommendation should have a clear reason.

Every empty state should have a clear next step.

## Recommendations Should Be Prioritized

The application should not present every possible idea at once.

The coach needs the highest-priority decisions first.

The Top Three Recommendation Rule should remain central to the experience.

## Missing Data Should Be Explained

Missing data should never feel like failure.

The app should say:

- What is missing
- Why it matters
- What improves when it is added
- Where to add it

## The Experience Should Stay Premium

The visual and interaction experience should remain:

- Modern
- Executive
- Minimal
- Calm
- Dark theme first
- Spacious
- Football operations centered

It should not become loud, cluttered, or gimmicky.
