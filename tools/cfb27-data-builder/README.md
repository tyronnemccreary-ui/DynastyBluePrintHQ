# CFB27 Data Builder

The CFB27 Data Builder is a utility workspace for preparing team data before it enters the Dynasty Blueprint HQ application.

It does not run inside the app. It does not implement OCR, AI vision, or screenshot parsing yet.

## Purpose

This builder will eventually convert reviewed team screenshots into structured CFB27 team records.

The output is intended to feed:

```text
database/teams/cfb27-teams.ts
```

The application should continue reading validated team records from the main team database architecture, not directly from screenshots.

## Current Manual Workflow

1. Place screenshots in `input/` for reference.
2. Manually review each screenshot.
3. Enter the extracted team values into JSON files in `output/`.
4. Use `team-extraction-template.json` as the shape for each team.
5. Run the builder script later to validate, normalize, and generate a combined JSON output.

Current template:

```text
tools/cfb27-data-builder/output/team-extraction-template.json
```

Current generated placeholder:

```text
tools/cfb27-data-builder/output/cfb27-teams.generated.json
```

## OCR/Vision Workflow

The extraction pipeline supports multiple modes so the app is not dependent on Apple Vision alone:

```text
apple-vision
tesseract
vision-json
manual-reviewed
```

Run a first-pass extraction on only 5-star teams:

```text
node --experimental-strip-types tools/cfb27-data-builder/src/extract-screenshot-data.ts --mode=vision-json --prestige=5
```

Mode behavior:

- `apple-vision`: reads existing `.ocr.txt` sidecar transcripts generated by the Swift helper.
- `tesseract`: attempts local Tesseract OCR if the `tesseract` command is installed, writes `.tesseract.txt`, and falls back to sidecars if unavailable.
- `vision-json`: reads externally generated reviewed values from `output/cfb27-teams.vision-reviewed.json`.
- `manual-reviewed`: reads manually reviewed values from `output/cfb27-teams.manual-reviewed.json`, then falls back to the Vision-reviewed JSON if the manual file is empty.

OCR and Vision output are still treated as candidate data. Human review remains required before importing generated records into the live app database.

The diagnostics report shows method, successful fields, failed fields, confidence, and source file:

```text
tools/cfb27-data-builder/output/extraction-diagnostics.json
```

Workflow:

1. Read screenshots from `input/`.
2. Detect screenshot category:
   - `team-overview`
   - `my-school-grades`
   - `dynasty-blueprint`
   - `ad-expectations`
3. Extract candidate values into JSON.
4. Require human review before accepting records.
5. Validate and normalize reviewed records.
6. Generate structured output for the main database.

## Screenshot Extraction Pipeline

Sprint 18 adds the first extraction pipeline shell:

```text
tools/cfb27-data-builder/src/extract-screenshot-data.ts
```

The pipeline scans screenshots from:

```text
tools/cfb27-data-builder/input/
```

Screenshots should be placed in prestige folders:

```text
5 Star/
4 Star/
3 Star/
2 Star/
1 Star/
```

The folder name determines `teamPrestige`.

Current extraction behavior:

- Image files are discovered from the prestige folders.
- Optional transcript sidecars can be placed next to screenshots using `.txt` or `.ocr.txt`.
- The script parses structured fields from those transcripts when present.
- Missing or uncertain values are set to `Not Available` where appropriate and flagged for review.
- The pipeline does not auto-trust OCR output.
- Human review is required before importing generated records into the live app database.

Generated outputs:

```text
tools/cfb27-data-builder/output/cfb27-teams.raw.json
tools/cfb27-data-builder/output/cfb27-teams.needs-review.json
tools/cfb27-data-builder/output/cfb27-teams.generated.ts
```

If no screenshots are present, the generated outputs will contain empty records.

Before extraction, place screenshots inside the matching prestige folder. The input count utility writes:

```text
tools/cfb27-data-builder/output/input-file-count-report.json
```

The expected screenshot total for the current CFB27 set is 117.

## Output Destination

Generated team data should eventually be copied, transformed, or imported into:

```text
database/teams/cfb27-teams.ts
```

The database files store facts only. Recommendation logic belongs in the Football Operations engine, not in this builder.
